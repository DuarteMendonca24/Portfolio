Spell Shaders

Project Status: Completed

Software used: Unity

Language used: HLSL & C#

About this project

This project consists of a package of shaders themed around "Spells," created for Unity's Built-in Render Pipeline. The development involved a variety of shaders, including surface, vertex, and fragment shaders, and employed a range of techniques such as post-processing, Fresnel effects, rim lighting, and depth textures.

This project was developed for the Advanced 3D Programming Techniques course by a team comprising myself, Ana Neves, and Francisco Andrade. We drew inspiration from "Harry Potter" and "Doctor Strange" to conceptualize some of the shaders.

I was personally responsible for developing the spells "Out of Body," "Shield," "Poison," and "Shadow World."

 Out of Body

Motivation and Context in a Game


The "Out of Body" shader was inspired by the astral projection effect in the film "Doctor Strange". This effect can be utilized in games with magical elements to simulate an astral projection spell or the death of enemies.

Effect Composition


The effect consists of a surface shader called “afterImage” that gives the character a transparent, ghostly appearance while maintaining the original texture. Additionally, the “JumpBack” script handles animations, and the “AfterImage” script creates multiple meshes of the character during the animation.

Interaction and Flexibility


The shader is designed for humanoid characters with animations but can be used on other objects as long as the difference in positions between the original object and the created images is visible. The color and transparency of the object can be modified, and the “AfterImage” script has a variable to control the number of meshes created.

Shader Theory


The shader uses properties for the object's textures (Albedo, Normal, Emission) and includes a color and two ranges to control color intensity and transparency. To enhance realism, the normal map and base color are applied according to predefined textures. Inspired by the diffuse light aura effect in "Doctor Strange", an Emission texture is used. A fresnel effect is added for a colored contour based on the defined color and intensity. The material's alpha is set to the inspector's declared alpha.

 Shield

Motivation and Context in a Game


The "Shield" shader was inspired by the common depiction of magical barriers in games and movies. This unlit shader can create a protective barrier effect and is versatile enough for use in both magical and sci-fi themed games.

Effect Composition


The effect is created using an Unlit Shader named “Shield,” which implements methods to produce a shield with distortion, edge glow, and a growing animation based on a range.

Interaction and Flexibility


This shader can be applied to a sphere and placed anywhere to simulate a shield. By adjusting the sphere's scale, the shield's size can be manipulated. The shader allows for customization of the normal map texture, distortion properties, and shield color.

Shader Theory


The shader defines properties for a normal map, distortion strength, speed, and direction, as well as colors and intensity for the fresnel effect and a range to control shield size.

Local and Screen Position Variables: Used for calculations related to shield growth and distortion.

Fresnel Effect: Applied for edge color. Smoothstep Function: Creates a smooth transition effect.

Clip Function: Discards pixels below a certain y-axis value, simulating a half-sphere shield.

Distortion Effect: Achieved by offsetting UV coordinates over time, controlled by distortion parameters. The normal map is sampled and adjusted, then added to screen coordinates for a dynamic visual effect.

Alpha Calculation: Based on the Fresnel effect, interpolating between edge glow and edge color for a smooth transition.

Dynamic Edge Glow and Distortion: If a pixel’s local y position is below a certain value, the shader returns a combination of edge color and projected texture, creating edge glow and dynamic distortion. Otherwise, the shader sets opacity to zero, enabling a shield growth effect.

 Poison

Motivation and Context in a Game


The idea for this shader emerged from a professor's suggestion to create a post-processing effect. This shader can simulate player poisoning in a game, adding an immersive visual element.

Effect Composition


The effect consists of an image effect shader named “poisonEffect,” which creates an oval vignette and generates noise. It also uses a script called “Post” to apply the material for post-processing effects.

Interaction and Flexibility


Since this effect uses an image shader, it interacts with the player's camera. The shader allows customization of the vignette size and fade effect, noise speed, and color. By changing the color to a reddish-orange, it can also simulate a fire effect.

Shader Theory


The shader was converted from GLSL to HLSL, based on a noise shader from "The Book of Shaders".

Noise Generation: The converted noise shader was animated using the _TimeScale property for a dynamic effect. Adjustments with smoothstep provided a fog-like, toxic appearance.

Vignette Creation: UV coordinates were normalized to center the screen. The distance from the center was calculated, and smoothstep was used for a smooth transition between the central image area and the edges.

Final Color Calculation: The vignette mask was applied to preserve the central image color, while the inverted mask, tint color, and noise created the vignette effect.

The final effect is rendered if the _IsActive toggle is enabled.

The “Post” script uses Graphics.Blit to apply the poison material for post-processing. It includes functions to switch between post-processing effects and deactivate them after 4 seconds, utilizing the _IsActive toggle.

 Shadow World

Motivation and Context in a Game


The motivation for this shader came from exploring the use of the camera's depth texture. This shader can simulate a spell where the player enters a shadow world, becoming invisible to others, similar to an ability in the game "Paladins."

Effect Composition


The effect consists of an image effect shader named “shadowEffect,” which interpolates the camera's depth texture with the main texture and a specified color. It also utilizes the previously mentioned "Post" script.

Interaction and Flexibility


Since this effect uses an image shader, it interacts with the player's camera. The effect is simple, allowing adjustments only to the color and the interpolation intensity.

Shader Theory


Properties: Declared properties allow changes to the effect's intensity and color.

Depth Texture: The global property _CameraDepthTexture is declared to access the camera's depth texture. The red channel value of this texture is accessed and converted to a range of zero to one using the Linear01Depth function.

Interpolation: The linear depth value is interpolated with the original texture color (col) using the specified intensity (_Intensity). If the _IsActive variable is enabled, a second interpolation mixes the original color with the result of the first interpolation, tinted by the specified color (_Color), using _Intensity as the blending factor.
